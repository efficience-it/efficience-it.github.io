# Documentation

This documentation follows the [Diataxis](https://diataxis.fr/) framework, organized into four categories based on user needs.

| Category | Purpose | Question it answers |
|----------|---------|---------------------|
| [Tutorials](tutorials/) | Learning-oriented | "How do I get started?" |
| [How-to guides](how-to-guides/) | Task-oriented | "How do I do X?" |
| [Reference](reference/) | Information-oriented | "What is X?" |
| [Explanation](explanation/) | Understanding-oriented | "Why does X work this way?" |

## Contributing

When adding documentation, pick the category that best fits the content:

- **Tutorials** walk a beginner through a complete experience from start to finish. They are lessons, not instructions.
- **How-to guides** are step-by-step directions for achieving a specific goal. They assume some existing knowledge.
- **Reference** is dry, accurate, and complete technical description (architecture, file structure, configuration options, API).
- **Explanation** provides context, background, and reasoning. It clarifies design decisions and trade-offs.
